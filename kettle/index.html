Rib Wrangling


This page demonstrates the basics of how a 3D scene is represented by Pixar's RenderMan scene description RIB file. Such RIB files are normally generated automatically by applications such as Houdini or Maya (plus Pixar's plugin). However, manually writing RIB files using a text editor focuses attention on the way that "elements" such as color, opacity, shading, lighting and coordinate systems are presented to Pixar's PhotoRealistic RenderMan rendering system.

Modelling Process

I struggled initially with this assignment, trying to grasp how this language functioned, but soon it became apparent that this was no python! This was more like HTML for 3D. After figuring that out it just became a process of try something, make shapes align, rinse repeat.

This section showcases how the modelling was handled using quadric surfaces. I created seperate rib Archives for the cups and the Teapot.


The Teapot is broken up into 5 main pieces, the "mainpot","top-lid","spout","handle" and "base". Each of these parts in turn are made up of 2 or more quadric surfaces

Lighting

For the lighting setup I used 2 rectangle Lights, one as the key light and the other as a rim light.
I then used a domelight with a HDR Image to get some fill, I set the intesnsity of the HDR to 0.5 so
that it was a little subdued

Shading

The scene uses mostly just two Archives for the material, one for the ground plane and one for the Teapot and Cups.

For the kettle and cup surface I wanted to try and emulate the surface of my first reference Image. So I tried using a fractal noise pattern
to drive the displacement on the surface of the kettle. This worked quite well, the tricky part was to try and get a subtle value without getting too noisy

I also used a grunge map, to try and add some variation in the specularity of the surface.
